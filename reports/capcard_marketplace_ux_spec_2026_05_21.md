# CapCard Marketplace UX Spec - 2026-05-21

The internal marketplace should make evidence useful before it makes anything public.

Recommended views:

- Candidate list: scan candidate status, blockers, and recommended action.
- Candidate detail: show evidence basis, limitations, and not-claimed boundaries.
- Evidence traceability tab: show ledgers, no-upload gates, and source rows.
- Limitations tab: keep bounded scope visible.
- Validator/stress-test tab: show latest validation and adversarial failures.
- Human review checklist: require reviewer, date, and decision before promotion.
- No-go boundary strip: PETAL false, HF false, public false, production false.
- Promotion history: audit state changes and commits.
- Command Center card view: internal status only, no deploy controls.

The mock index contains:

- EML Puzzle Evidence Kernel: strong internal candidate.
- Qwen Puzzle Curriculum Pack: blocked with exact blockers.

This is not a production marketplace asset.
